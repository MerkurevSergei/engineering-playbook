## Principles

- Publish reusable engineering artifacts, not raw private notes.
- Prefer checklists, templates, decision guides, and short implementation cards.
- Keep one primary home for each topic and cross-link related topics.
- Separate private source material from public synthesis.
- Make each note useful in a real engineering conversation, review, or design session.
