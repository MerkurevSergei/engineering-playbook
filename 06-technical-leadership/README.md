# Technical Leadership

This section covers technical decision-making, engineering standards, documentation culture, technical debt, team health, mentoring, and operational leadership.

## Artifacts

## Use This Section When

- A technical problem needs business framing.
- A team needs clearer standards or decision-making rules.
- Documentation is scattered, stale, or separated from delivery.
- Technical debt is creating visible cost but lacks a repayment argument.

## Output Standard

A leadership artifact is ready when it helps teams make better decisions, communicate trade-offs, and improve delivery without hiding engineering reality.
