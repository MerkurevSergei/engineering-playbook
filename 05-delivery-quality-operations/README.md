# Delivery, Quality & Operations

This section covers testing strategy, CI/CD, deployment readiness, observability, production operations, and incident learning.

## Artifacts

- [Testing strategy](testing-strategy.md)
- [Observability checklist](observability-checklist.md)

## Use This Section When

- A service is moving from implementation to release.
- Test scope is too slow, flaky, or unclear.
- Teams need production signals before users find problems.
- Deployment, rollback, monitoring, or incident ownership is not explicit.

## Output Standard

A delivery artifact is ready when it defines confidence signals, release gates, operational visibility, and recovery paths.
