# Data, Integration & Reliability

This section covers API contracts, messaging, asynchronous workflows, data consistency, idempotency, retries, and integration reliability.

## Artifacts

- [Async patterns map](async-patterns-map.md)

## Use This Section When

- A system communicates through APIs, events, queues, files, CDC, or ETL.
- A workflow crosses service or team boundaries.
- Failure handling, retries, duplication, ordering, or consistency are unclear.
- Kafka, REST, OpenAPI, outbox, idempotency, or DLQ decisions need explicit framing.

## Output Standard

An integration artifact is ready when it explains the contract, ownership, delivery guarantees, failure behavior, observability, and recovery process.
