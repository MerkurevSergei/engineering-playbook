# Architecture & Design

This section helps turn requirements and constraints into explicit architecture decisions.

## Artifacts

- [ADR template](adr-template.md)
- [Quality attributes checklist](quality-attributes-checklist.md)

## Use This Section When

- A decision has long-term consequences or multiple reasonable options.
- A design depends on non-functional requirements such as reliability, latency, scalability, security, or operability.
- Teams need a shared explanation of why a choice was made.
- A system boundary, integration style, or data ownership model is under discussion.

## Output Standard

An architecture artifact is ready when it names the decision, explains the context, compares realistic options, states consequences, and provides evidence or validation criteria.
